---what is docker or container? ---
Docker is a tool that lets you package an application with everything it needs to run (code, libraries, dependencies, environment) into a single unit called a container.

---Docker Container---
• like it a bundle where we pack all necessary data of an app into single thing, that is called as containers.
• now it would be more easies to transfer app from one device to another (irrespective it is linux, windows, macOS).

---> properties---> 1) portable (can share to any devices).
                    2) it is lightweight in nature (very less overhead, simple to create/delete, size is very low).
                    3) if my device is some node x version and i wanna build the two app with d/f version then on single device it is not possible, so i can use is docker, make two container with d/f version i can use those container with according to my app.

---Port binding---
Command: docker run -p 8080:3306 IMAGE_NAME
    Note: Maps container port to host port
    Format: -p HOST_PORT:CONTAINER_PORT
    Example: docker run -p 8080:3306 mysql
Explanation: 
• cant use one port with two or more container
• HOST_PORT (8080): Port on your machine to access the service
• CONTAINER_PORT (3306): Port inside the container where service runs
• Allows external access to containerized services
• Multiple port bindings: docker run -p 8080:3306 -p 33060:33060 IMAGE_NAME
